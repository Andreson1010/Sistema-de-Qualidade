# Projeto 6 - Versionamento e Controle de Dados em Pipelines CI/CD com Github Actions e Kubernetes

# O Act é somente para teste local do GitHub Actions.

# Se for executar diretamente no GitHub Actions (sem usar o Act local), você pode simplificar o workflow, não precisando usar a imagem catthehacker/ubuntu:act-latest, pois o ambiente padrão do GitHub já fornece tudo que você precisa.

# Um workflow recomendado seria assim:

name: DSAProjeto6

on:
  push:
    branches:
      - main
    paths:
      - 'modelo/**'
      - 'treinamento/**'
      - 'appdsa.py'
      - 'requirements.txt'
      - 'Dockerfile'

jobs:
  build-test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout do código
        uses: actions/checkout@v4

      - name: Configurar Python
        uses: actions/setup-python@v5
        with:
          python-version: 3.12

      - name: Instalar dependências Python
        run: |
          pip install --upgrade pip
          pip install -r requirements.txt

      - name: Treinar modelo
        run: python treinamento/dsa_treina_modelo.py

      - name: Login no Docker Hub (opcional, necessário apenas se for enviar a imagem)
        uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}

      - name: Construir imagem Docker com novo modelo
        run: docker build -t dsa-p6-app:latest .

      - name: Validar manifestos Kubernetes
        uses: stefanprodan/kube-tools@v1
        with:
          command: kubeconform -summary -strict k8s/

# Observações adicionais:

# Caso queira publicar a imagem Docker no Docker Hub ou GitHub Container Registry, adicione a etapa apropriada.

# Lembre-se: Ao executar com o GitHub, seus arquivos vão para a internet e na versão gratuita do GitHub, para um repositório público.
